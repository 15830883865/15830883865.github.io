单例模式

***计模式**是在大量的实践中总结*和理论化之后优选的代码结构、编程风格、以及解决问题的思考方式。设计模式免去我们自己再思考和摸索。就像是经典的棋谱，不同的棋局，我们用不同的棋谱。”套路”

所谓类的单例设计模式，就是采取一定的方法保证在整个的软件系统中，对某个类****只能存在一个对象实例****，并且该类只提供一个取得其对象实例的方法。

如果我们要让类在一个虚拟机中只能产生一个对象，我们首先必须将类的构造器的访问权限设置为private，这样，就不能用new操作符在类的外部产生类的对象了，但在类内部仍可以产生该类的对象。因为在类的外部开始还无法得到类的对象，只能调用该类的某个静态方法以返回类内部创建的对象，静态方法只能访问类中的静态成员变量，所以，指向类内部产生的该类对象的变量也必须定义成静态的。

static关键字的使用

  1. static修饰属性

    静态属性在内存空间中只有一份，在类中所有对象所共享。每个对象都有一份实例变量。

    jdk6及之前存放在方法区，jdk7及以后存放在堆空间。

    可以被类直接调用，也可以被对象调用

  2. static修饰方法

    随着类的加载而加载

    可以通过“类.静态方法”的方式，直接调用静态方法

    静态方法内可以调用静态的属性或静态的方法

  3. 开发中，什么时候需要将属性声明为静态的？

    判断 当前类的多个实例是否能共享此成员变量，且此成员变量的值是相同
